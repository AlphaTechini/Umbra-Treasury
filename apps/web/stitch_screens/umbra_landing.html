Private Treasury.
Selective Transparency.

DAO transactions are private but can be revealed when needed. Built for institutional capital requiring rigorous operational security and verifiable compliance on public ledgers.

visibility The Vulnerability

Public blockchains expose everything.

Standard operational structures leave strategic capital vulnerable. Broadcasting every transaction payload, recipient address, and treasury balance to a public ledger compromises institutional security and negotiation leverage.

vpn_key The Protocol

Private by default, reveal when needed.

Execute transactions within a shielded pool. Maintain complete operational privacy while retaining the cryptographic capability to generate granular viewing keys for auditors, compliance entities, or community stakeholders.

Execution Flow

Cryptographic process for selective disclosure.

01 lock

Private transactions

Funds are moved within the shielded protocol. Sender, receiver, and amount remain cryptographically hidden from public network scanners.

02 assignment_returned

Request disclosure

Authorized auditors, DAO governance proposals, or regulatory bodies submit on-chain requests for specific transaction blocks or operational timeframes.

03 key

Reveal with viewing key

Treasury operators sign a message to generate a zero-knowledge viewing key, selectively decrypting only the requested data vectors.
