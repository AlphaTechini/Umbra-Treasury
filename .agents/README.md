# Agent Memory

## Purpose

This folder stores project-level memory for implementation decisions, constraints, and tradeoffs that should survive across coding sessions.

## Architectural Decisions And Tradeoffs

The project keeps implementation memory separate from the public product docs so future work can quickly recover agreed decisions without rereading every planning file.

The tradeoff is that some decisions are summarized twice: once in user-facing docs such as [Schema.md](file:///C:/Hackathons/Umbra%20SDK/Schema.md), and once in the agent guide. The guide should stay concise and point back to source docs when details matter.
